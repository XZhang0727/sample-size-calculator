## Bug修复：normInv函数错误导致所有计算结果错误 (13:38)
- 发现根本问题：原始 Beasley-Springer-Moro normInv算法在 p>0.92 时走错分支
- 当 p=0.975 时（α=0.05双侧对应的分位点）错误计算出 z=5.01 而非正确的 1.96
- 导致所有样本量结果约放大 6.5 倍（如横断面均值题应得255却显示1666）
- 修复：替换为 Peter Acklam 高精度算法（最大误差<1.15e-9）
- 验证通过：z(0.975)=1.9600, z(0.80)=0.8416等所有关键z值完全正确
- 已推送到 GitHub 并重新部署到 Vercel
